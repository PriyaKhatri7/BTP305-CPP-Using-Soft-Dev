Part A - Introduction

Across Translation Units

Workshop 1


In this workshop, you implement linkage, storage duration, namespaces, guards, and operating system interfaces.


Learning Outcome

Upon successful completion of this workshop, you will have demonstrated the abilities to

  • link to a variable in another translation unit
  • declare a local variable that lasts the lifetime of the program
  • guard a class definition from repetition
  • define a class within its own namespace
  • pass arguments to a program from the command line
  • describe to your instructor what you have learned in this workshop

Specifications

This workshop consists of three modules:

  • w1
  • Process
  • CString

CString Module

Code a class named CString that encapsulates a C-style null-terminated string.  Include separate header and implementation files for your module.  Wrap your header file in a conditional pre-processor macro that prevents repeated access.  Embed your class definition and its implementation in namespace sict.  In your header file, alongside your class definition, define an unmodifiable variable named MAX_CHAR that holds the number of characters to be stored in a CString object excluding the null byte.  Set the value of that variable to 14.

Include in or with your class definition:

  • a constructor that receives optionally the address of an unmodifiable C-style null-terminated string and stores up to the maximum number of characters specified.  Your constructor defaults to an empty string and checks for receipt of the null address.  Yourr constructor stores an empty string if the address received is the null address.
  • a member query named display()that receives a reference to an ostream object and inserts into that object the string stored in your CString object. 
  • a non-friend helper operator that inserts the stored string into the left ostream operand of an insertion expression.  This operator prefaces the string with an insertion counter and increments that counter.  Before the very first string only, this operator inserts a one-line heading as shown below.  The initial value for the counter is set outside your module.

Do not use the string class of the standard library in this workshop.  You may use the cstring functions.

Include in your implementation file:

  • the definition of a global variable that holds the initial value of the record counter. 

Process Module

The Process module consists of a single function that receives the address of a C-style null-terminated string, creates a CString object using the string, and inserts the object into the output stream. 

You can find copies of the header and implementation files for this module here and here respectively.

w1 Module

Code the implementation file for a main() function that receives the name of an input file as a command-line argument.  Your function echoes the command-line input, checks for the correct number of command-line arguments, and if successful, attempts to open the named file for reading. 

If successful, your function reads a record from the file.  If your function has read the record successfully, it passes the address of the record to the process function of the Process module.  Your function continues extracting records from the file until it has read all of them.  You may assume that any record in any file available will never contain more than 1000 characters.

If the user has not entered enough arguments, your function inserts an error message into the error stream and terminates execution.  If the user has entered too many arguments, your function inserts an error message into the error stream and terminates execution.  If your function fails to open the named file for reading, your function inserts an error message into the error stream and terminates execution.  Sample error messages are shown below: 

 Command Line : w1

 *** Insufficient number of arguments ***
 Usage: w1 filename

 Command Line : w1 file1.dat file2.dat

 *** Too many arguments ***
 Usage: w1 filename

 Command Line : w1 file1.dat

 *** Failed to open file file1.dat ***

Define a global variable that holds the initial value of the record counter and initialize that variable to the value 10.
